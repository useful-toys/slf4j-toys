## Key Features

### Documentation Quality
- **"No Inventions" rule**: All documentation must be verifiable from actual code or official docs
- **README synchronization**: API changes require README updates
- **TDR structure**: Standardized format for technical decisions

### AI Tracking
- **@AIGenerated annotation**: Track AI-generated code
- **Co-authorship**: Include AI in commit messages for transparency

### Testing Standards
- **Coverage target**: >90% code coverage
- **Assertion messages**: Required "should..." format
- **Test organization**: @Nested classes and @DisplayName

### Adaptations for slf4j-toys
- Java 8+ compatibility (vs Java 21 only in test-mock)
- Multi-module structure acknowledged
- Broader Lombok usage documented
- Pragmatic Javadoc policy (no redundant @Override docs)

## Benefits

1. **Consistency**: Same structure as slf4j-test-mock for familiar workflow
2. **Quality**: "No Inventions" prevents incorrect documentation
3. **Transparency**: AI tracking and co-authorship
4. **Standards**: Clear testing and documentation expectations
